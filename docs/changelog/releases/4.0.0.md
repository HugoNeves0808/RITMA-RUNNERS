# Release 4.0.0

This file keeps the implementation history and release notes being prepared for the fourth major project release.

## Release Summary

### Current preparation scope

- Scope definition for phase 4 starts after the completed `3.0.0` release closure.
- New implementation notes should be added here as the next release work is delivered.

## Implemented So Far

### Custom trainings module

- Added a new `Treinos` area to the web navigation and routing.
- Added authenticated backend endpoints under `/api/trainings` for table data, create options, filters, CRUD, completion toggling, delete scope, and custom training-type management.
- Added Flyway migration `V18__add_custom_trainings.sql` to persist custom trainings, recurring series metadata, and user-managed training types.
- Added recurring-series creation with interval, until-date, and weekday selection.
- Added single-item delete and whole-series delete flows.
- Added web grouping that places trainings between race dates instead of using manual race association in the UI.
- Added visual handling for past race blocks, stale planned trainings, and bulk completion updates from the page header warning.

### Current UX notes

- The `Treinos` page is explicitly marked as in development.
- The backend still accepts `associatedRaceId`, but the current web flow sends `null` and organizes trainings by dated race blocks instead.
- Editing a recurring training currently updates only the selected row, not the whole generated series.
