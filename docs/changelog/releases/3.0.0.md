# Release 3.0.0

Release status: `completed`

Release date: `2026-04-17`

This file keeps the implementation history and release notes for the third major project release.

## Release Summary

### Release scope

- Finalized the third major project release around localization polish, shared management-flow consistency, and UI cleanup across the web and mobile clients.
- Closed phase 3 after tightening the authenticated language experience, race-management wording consistency, and a series of smaller UX fixes discovered during final release validation.

## Implementation Notes

### Localization and translated UI across web and mobile

- added English and Portuguese translations for the `Profile` page
- localized the profile summary labels, empty states, and fallback errors
- updated shared race-action wording from `Click to Update` to `Update Status` / `Atualizar Estado`
- continued PT/EN copy cleanup and small authenticated UI consistency fixes
- extended the same localization pass into the mobile auth, admin, profile, and settings flows so both clients now use the same updated terminology
- aligned race-type presentation across web `Races`, `Best Efforts`, and `Podiums` so translated default race-type names stay consistent in lists, filters, and details views

### Race details and status update flow

- race details now surface the past-race status warning directly inside the details view when a race still needs a final status update
- the warning action in race details now uses the same red action pattern as the race cards and opens edit mode immediately
- adjusted the placement of the status-update action so it sits beside the status badge in the details card
- aligned race cards so the status badge appears first and the status-update action appears to its right

### Calendar and best efforts behavior

- restored race edit and delete flows from calendar mode by wiring the calendar details drawer to the same edit/delete flow used by the table view
- calendar race details now support opening the edit drawer, deleting races, and refreshing visible data after changes
- replaced the central `Best Efforts` race-type management modal with the shared side-panel management flow used elsewhere in the app
- refreshed best-efforts data and available race-type filters after side-panel management closes
- fixed the web `Best Efforts` build after that side-panel migration so the release finishes with the shared management flow working cleanly again

### Final phase-3 polish before release closure

- removed the old public `Future Goals` page and its login-screen link, keeping the project focused on the authenticated product flows already delivered
- simplified the web login side panel by removing the lower development callout and divider
- simplified the web `Settings` preferences area by removing the old `Remember me` section and keeping language as the single saved local preference
- split the web language behavior between a saved default preference and an in-session override, so the sidebar language switch survives refresh inside the session but returns to the saved default after logout/login
- localized the request-account success toast in the frontend instead of depending on backend response copy, keeping PT/EN feedback correct in the login modal
- removed the extra `Request time` column from the web admin `Overview` pending-approvals preview table
- fixed the web `Podiums` filter counter so only `Tipo de pódio` shows the number of selected filter options instead of rendering per-option totals beside each checkbox
- standardized PT copy to prefer `Circuito` instead of `Percurso` in the race form and related race-detail labels
