# Release 3.0.0

Release status: `in preparation`

Release date target: `TBD`

This file keeps the implementation history and release notes being prepared for the third major project release.

## Release Summary

### Current preparation scope

- Web localization and UI consistency improvements across authenticated pages
- Race status follow-up UX refinements in list, calendar, and race-details flows
- Best-efforts filter management alignment with the shared side-panel pattern

## Implementation Notes

### Localization and translated UI

- added English and Portuguese translations for the `Profile` page
- localized the profile summary labels, empty states, and fallback errors
- updated shared race-action wording from `Click to Update` to `Update Status` / `Atualizar Estado`
- continued PT/EN copy cleanup and small authenticated UI consistency fixes

### Race details and status update flow

- race details now surface the past-race status warning directly inside the details view when a race still needs a final status update
- the warning action in race details now uses the same red action pattern as the race cards and opens edit mode immediately
- adjusted the placement of the status-update action so it sits beside the status badge in the details card
- aligned race cards so the status badge appears first and the status-update action appears to its right

### Calendar and best efforts behavior

- restored race edit and delete flows from calendar mode by wiring the calendar details drawer to the same edit/delete flow used by the table view
- calendar race details now support opening the edit drawer, deleting races, and refreshing visible data after changes
- replaced the central `Best Efforts` race-type management modal with the shared side-panel management flow used elsewhere in the app
- refreshed best-efforts data and available race-type filters after side-panel management closes
